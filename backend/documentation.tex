\section{Project Overview}
Geo Serbia is a Serbia-focused geography game inspired by GeoGuessr. A player sees a location image and must click on a map to guess where the place is. The system computes distance and score, tracks progress, and presents leaderboards and profile statistics.

The final system consists of:
\begin{itemize}[noitemsep]
  \item Backend: FastAPI + Tortoise ORM + PostgreSQL
  \item Frontend: Vue 3 + Vue Router + Axios + Leaflet
  \item Authentication: cookie-based JWT sessions (HTTP-only cookies)
  \item Adaptive gameplay: user and location difficulty profiling
\end{itemize}

\section{Goals and Scope}
\subsection{Primary Goals}
\begin{itemize}[noitemsep]
  \item Build a game-like UX, not CRUD-like pages
  \item Support secure user and admin roles
  \item Add adaptive difficulty based on player performance
  \item Provide leaderboard and profile analytics
\end{itemize}

\subsection{Out of Scope}
\begin{itemize}[noitemsep]
  \item Full machine-learning recommendation engine
  \item Real-time multiplayer gameplay
  \item Mobile-native app implementation
\end{itemize}

\section{System Architecture}
\subsection{Backend Modules}
\begin{itemize}[noitemsep]
  \item \texttt{src/domains/users}: register/login/logout/me/profile
  \item \texttt{src/domains/locations}: creation, approval flow, list/filter
  \item \texttt{src/domains/games}: session, rounds, adaptive selection
  \item \texttt{src/domains/admin}: global and adaptive admin analytics
\end{itemize}

\subsection{Frontend Modules}
\begin{itemize}[noitemsep]
  \item Auth page with login/register toggle and session restore
  \item Home hero page and daily challenge call-to-action
  \item Game page (image + map split screen)
  \item Summary, leaderboard, profile, add location, admin pages
\end{itemize}

\section{Data Model}
\subsection{Core Game Tables}
\begin{itemize}[noitemsep]
  \item \texttt{game\_sessions}: daily user sessions
  \item \texttt{rounds}: per-guess records (distance, score)
  \item \texttt{locations}: approved/pending locations
  \item \texttt{users}: accounts and admin flag
\end{itemize}

\subsection{Adaptive Tables}
\begin{itemize}[noitemsep]
  \item \texttt{user\_skill\_profile}
  \item \texttt{location\_difficulty\_profile}
  \item \texttt{adaptive\_decision\_log}
\end{itemize}

\section{Authentication and Authorization}
\begin{itemize}[noitemsep]
  \item Login sets access and refresh tokens in HTTP-only cookies
  \item Protected endpoints require valid cookie token
  \item Admin endpoints require \texttt{is\_admin = True}
\end{itemize}

\section{Adaptive Difficulty Design}
\subsection{User Skill Profile}
For each user, the system keeps:
\begin{itemize}[noitemsep]
  \item \texttt{skill\_rating}
  \item \texttt{recent\_avg\_distance\_km}
  \item \texttt{recent\_avg\_points}
  \item \texttt{consistency\_index}
\end{itemize}

\subsection{Location Difficulty Profile}
For each location, the system keeps:
\begin{itemize}[noitemsep]
  \item \texttt{difficulty\_rating}
  \item \texttt{global\_avg\_distance\_km}
  \item \texttt{global\_avg\_points}
  \item \texttt{attempt\_count}
\end{itemize}

\subsection{Challenge Start}
\texttt{POST /api/v1/game/start?mode=adaptive|fixed}
\begin{itemize}[noitemsep]
  \item Adaptive: picks locations close to user skill target
  \item Fixed: random selection
  \item Logs decision and fallback usage
\end{itemize}

\subsection{Round Play}
\texttt{POST /api/v1/game/play}
\begin{itemize}[noitemsep]
  \item Computes distance and score
  \item Updates user and location adaptive profiles
  \item Returns score data and adaptive metadata
\end{itemize}

\section{Location Moderation Workflow}
\begin{itemize}[noitemsep]
  \item Regular users submit pending locations (\texttt{is\_approved=False})
  \item Admin-created locations auto-approve in service layer
  \item Admin can approve/reject pending submissions
\end{itemize}

\section{Seeder Strategy}
Seeders create deterministic gameplay quality:
\begin{itemize}[noitemsep]
  \item Users: admin + personas (\texttt{tojaga}, \texttt{somi}, \texttt{maleza})
  \item Persona skill profiles:
  \begin{itemize}[noitemsep]
    \item \texttt{tojaga}: low skill (easy tier)
    \item \texttt{somi}: medium skill (normal tier)
    \item \texttt{maleza}: high skill (hard tier)
  \end{itemize}
  \item Locations: approved location set with spread difficulty profiles
\end{itemize}
